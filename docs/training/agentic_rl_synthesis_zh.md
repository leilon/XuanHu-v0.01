# Agentic-RL 合成数据方案说明

## 1. 为什么不能只靠公开医疗偏好数据

医疗 Agent 真正难的地方，不是“把一句答案写得像医生”，而是：

- 会不会先追问关键病史
- 会不会识别红旗症状
- 会不会根据场景调用工具
- 会不会区分门诊、急诊和收治风险
- 会不会把建议和证据对应起来

大多数公开 preference 数据只能监督“回答文本”，很难监督完整问诊行为，所以我们必须自己合成更贴近任务的数据。

## 2. 我们要合成什么样的数据

最终目标是得到适合 DPO / ORPO / PPO 类优化的样本：

- `prompt`
- `chosen`
- `rejected`
- `meta.case`
- `meta.failure_mode`
- `meta.judge`

其中：

- `chosen` 是更符合真实问诊流程的多轮轨迹
- `rejected` 不是胡说八道，而是“看起来还像回事，但关键步骤做错了”的轨迹

## 3. 合成流程

### 第一步：从种子问题构造隐藏病例包

输入可以是用户真实问题，例如：

- “我发烧咳嗽两天，需要吃什么药？”

但 teacher 不直接回答，而是先生成一个隐藏病例包：

- 主诉
- 年龄、性别
- 隐藏病史
- 必问项目
- 危险信号
- 建议工具
- 建议检查

这里有个关键升级：病例包必须遵循真实首程问诊逻辑，而不是平铺一些槽位。

比如呼吸道症状病例，必须优先包含：

- 起病时间
- 最高体温
- 咳嗽性质
- 有无胸闷气促
- 流行病接触史
- 基础病

如果是腹痛病例，病例包中应优先围绕：

- 部位
- 性质
- 放射痛
- 进食和排便关系
- 呕吐/腹泻/黑便/便血
- 必要时月经和妊娠相关信息

### 第二步：生成专家轨迹 `chosen`

teacher 依据病例包生成“更像真实医生”的首程问诊轨迹。

这条轨迹至少要体现：

- 从主诉开始按时间线追问
- 先问关键病史再给建议
- 对高风险场景及时升级
- 必要时调用工具
- 最后给出初步检查和分诊建议

### 第三步：生成失败轨迹 `rejected`

失败轨迹必须对应明确的 failure mode，而不是做成明显错误的废答案。

重点 failure mode：

- `premature_answer`
  - 还没问清病史就直接开药
- `missed_red_flag`
  - 出现危险信号却没有让用户急诊
- `no_tool_use`
  - 明明需要看报告或查药物禁忌，却直接拍脑袋回答
- `no_grounding`
  - 给建议但没有证据或边界
- `missed_history`
  - 漏问过敏史、基础病、长期用药、妊娠相关信息

### 第四步：judge 打分并筛选

只有当 `chosen` 明显优于 `rejected` 时才保留。

评分维度：

- `safety`
- `coverage`
- `grounding`
- `tool_use`
- `overall`

这一步很重要，因为没有 judge 的话，很容易把大量“差异很小”或“标签方向不稳”的样本混进训练集。

## 4. 临床首程问诊逻辑必须注入合成流程

这是这版方案最重要的变化之一。

### 4.1 症状优先

合成出来的轨迹必须从当前主诉出发，而不是上来就问一堆通用槽位。

例如：

- 发热咳嗽先问体温、咳嗽性质、气促、流行病史
- 腹痛先问部位、性质、进食关系、排便情况
- 胸痛先问放射痛、活动关系、呼吸困难和大汗

### 4.2 条件化问诊

问诊逻辑必须条件化：

- 男性不优先追问是否怀孕
- 育龄女性在相关场景下需要追问月经和妊娠可能
- 有长期记忆的老用户，需要优先结合既往病史和既往类似发作

### 4.3 分诊和收治判断

`chosen` 里不只是“回答对”，还要体现分诊水平：

- 哪些问题普通门诊可处理
- 哪些问题要尽快门诊
- 哪些问题要急诊
- 哪些问题要由急诊进一步判断是否收治

## 5. 三个具体例子

### 例子一：发热咳嗽 + 育龄女性

**种子问题**

- “我发烧咳嗽两天，需要吃什么药？”

**隐藏病例包**

- 29 岁女性
- 最高体温 39.1℃
- 对布洛芬过敏
- 有妊娠可能

**chosen**

- 先问发热多久、最高体温、咳嗽有无痰、是否气促
- 再问过敏史、当前用药
- 因为是育龄女性且后续可能涉及用药，补问末次月经和妊娠可能
- 调用药品知识和指南工具
- 给出初步检查和是否需要发热门诊评估

**rejected**

- 直接推荐布洛芬退热
- 没问过敏史
- 没问妊娠可能

### 例子二：胸痛 + 呼吸困难

**种子问题**

- “我下午突然胸口闷痛，还有点喘不上气。”

**chosen**

- 先确认持续时间、疼痛性质、是否放射、是否伴大汗
- 立即建议急诊
- 不把问题拖成普通门诊问答

**rejected**

- 说可能是焦虑或疲劳，建议先休息观察

### 例子三：老用户 + 既往糖尿病 + 再次腹痛

**种子问题**

- “我这两天又肚子痛，吃不下饭。”

**长期记忆**

- 既往 2 型糖尿病
- 曾因胃肠炎样症状就诊
- 长期口服二甲双胍

**chosen**

- 先问本次腹痛与上次相比是否加重
- 追问呕吐、腹泻、脱水、血糖控制、近期饮食
- 结合长期用药和基础病判断脱水/代谢紊乱风险
- 给出门诊还是急诊评估建议

**rejected**

- 完全不引用既往病史
- 把这次当成新用户对待，从头问一遍

## 6. 对 7B 的训练建议

如果主模型是 `Qwen2.5-7B-Instruct`，推荐节奏是：

1. 先用高质量医疗 SFT 把“会问诊、会分诊、会做检查建议”的基本功训稳
2. 再加入 Agentic-RL 数据去优化行为顺序和安全分层

数据配比建议：

- `60%` 高质量医疗 SFT
- `25%` 合成 agentic preference
- `15%` 过滤后的公开 reward / preference 数据

原因是 7B 先要学会“像一个靠谱的医疗助手一样工作”，再去学“在多个合理路径里选更优的那个”。
