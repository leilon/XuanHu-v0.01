# MedAgent 主体架构设计

## 1. 项目目标

项目目标不是做一个只会答题的医疗问答模型，而是做一个贴近真实互联网医疗首程问诊流程的多 Agent 系统。它需要覆盖：

- 诊前：症状采集、危险信号识别、是否值得线下就诊
- 诊中：分诊、建议首批检查、结合报告和药品知识做辅助判断
- 诊后：问药、报告解释、复诊和长期记忆跟踪

核心能力包括：

- 多 Agent 协同
- 工具调用和 RAG 证据引用
- 基于 QLoRA 的 7B 持续学习
- 长期记忆驱动的个体化问诊
- 面向多轮问诊的 Agentic-RL 优化

## 2. 系统角色分工

### Orchestrator

负责任务编排和路由决策：

- 先识别用户意图是“首程问诊”还是“医学科普/知识解释”
- 判断当前问题是否需要先做首程问诊
- 判断是否插入报告解析
- 判断是否需要先做分诊再给建议
- 最后把各子 Agent 输出汇总成用户可读答案

### Intake Agent

负责首程问诊，不直接下结论，而是先做临床信息采集。它遵循真实问诊顺序：

1. 从当前主诉开始
2. 围绕主诉做症状时间线追问
3. 从主诉可能关联的疾病方向扩展问诊
4. 再补充一般病史、饮食作息、流行病史
5. 最后给出初步检查建议

### Triage Agent

负责分诊分层，区分：

- 普通门诊可处理
- 24 小时内专科/发热门诊就诊
- 需要急诊评估
- 需要急诊评估并考虑留观/收治

### Medication Agent

负责用药安全提醒，重点看：

- 过敏史
- 长期用药
- 潜在药物相互作用
- 育龄女性妊娠相关限制

### Report Agent

负责检验报告、化验单、影像结果的结构化解读。先做报告内容抽取，再把异常项和用户症状联系起来，而不是只做静态翻译。

### Education Agent

负责医学科普、检查指标解释、疾病概念解释。它和首程问诊分开，不应该拿长篇科普回答去训练首程问诊 Agent。

### RAG / Tools / Safety

- `RAG`：检索指南、药典、科普和流程文档
- `Tools`：药物相互作用、报告解析、后续可扩展检查建议工具
- `Safety Guard`：高危症状拦截和安全兜底

## 3. 首程问诊的临床逻辑

系统把“真实医生的首程问诊思路”作为基础模板，而不是泛泛地问几个槽位。

### 3.1 问诊顺序

首程问诊默认顺序如下：

1. 主诉和起病过程
2. 症状演变和严重程度
3. 与主诉相关的伴随症状
4. 从该主诉最常见的疾病方向继续追问
5. 既往基础病、手术史、过敏史、当前用药
6. 饮食、作息、吸烟饮酒
7. 流行病学接触史、旅行史
8. 在相关场景下追问冶游史/性接触史
9. 给出初步检查建议

### 3.2 条件化问诊

不是所有人都问同样的问题。

- 男性患者不会优先追问是否怀孕
- 育龄女性只有在与症状、用药或检查相关时，才重点追问月经和妊娠可能
- 老用户优先读取长期记忆中的慢病、过敏史、既往类似发作和既往检查结果
- 已知过敏史和长期用药不重复从零问，而是确认是否有变化

### 3.3 门诊与收治的判断

系统不替代线下医生做最终收治决定，但会把问题分层：

- `普通门诊/居家观察`
  - 症状轻、无明显危险信号、无高风险基础病失代偿表现
- `尽快门诊`
  - 需要 24 小时内专科评估，但暂不符合明确急救指征
- `急诊`
  - 胸痛、呼吸困难、意识障碍、抽搐、明显出血等危险信号
- `急诊评估并考虑留观/收治`
  - 持续高热不退、明显气促、反复呕吐不能进食、黑便/便血、严重基础病背景下病情加重

## 4. 长期记忆怎么接进系统

长期记忆不是“把用户所有历史对话都喂给模型”，而是结构化成几类临床价值最高的信息：

- 基础信息：年龄、性别
- 慢病：高血压、糖尿病、哮喘等
- 过敏史：药物、食物、既往严重不良反应
- 长期用药：胰岛素、阿司匹林、激素等
- 既往评估：上次相似症状的判断和建议

这些信息在本轮问诊中会影响：

- 追问顺序
- 风险分层
- 用药建议
- 是否需要更快线下就诊

## 5. 训练层怎么对齐这个架构

### SFT

SFT 需要覆盖的不只是“医学知识问答”，还包括：

- 首程问诊轨迹
- 科普解释轨迹
- 主诉到追问的展开逻辑
- 分诊结论
- 检查建议
- 报告解读
- 结合长期记忆的个体化表达

### Agentic-RL

Agentic-RL 的目标不是让模型回答更像书面答案，而是优化：

- 是否先追问再回答
- 是否问对关键病史
- 是否漏掉危险信号
- 是否在需要时调用工具
- 是否把门诊/急诊/收治建议分层讲清楚

## 6. 推荐面试讲法

一句话版本：

“我把这个医疗 Agent 设计成贴近真实临床首程问诊流程的多 Agent 系统，先由 Intake Agent 做症状优先的结构化问诊，再由 Triage Agent 做门诊、急诊和收治风险分层，并通过长期记忆注入既往病史和过敏史，最后用 RAG 和工具保证回答有证据、有边界、可落地。”
